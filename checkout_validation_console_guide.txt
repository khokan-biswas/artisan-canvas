# CHECKOUT FORM VALIDATION - CONSOLE LOG GUIDE

## Browser Developer Console Instructions:
1. Open the application at http://127.0.0.1:5175/
2. Navigate to the Checkout page
3. Press F12 to open Developer Tools
4. Go to the 'Console' tab
5. Fill in or interact with the checkout form

## Console Log Points in Checkout.jsx

### Log #1 - Line 188: Form Values Being Validated
Location: validateCheckoutFormSync() function
`
console.log("Validating form with:", { firstName, lastName, email, phone, address, city, state, zipCode, country });
`

WHEN IT FIRES:
- Every time the form validation runs (when you click PayPal, COD, or Indian Gateway buttons)
- When you try to submit any payment method

EXPECTED OUTPUT IN CONSOLE:
`
Validating form with: {
  firstName: "John",
  lastName: "Doe", 
  email: "john@example.com",
  phone: "9876543210",
  address: "123 Street",
  city: "New York",
  state: "NY",
  zipCode: "10001",
  country: "United States"
}
`

### Log #2 - Line 221: Validation Errors Found
Location: validateCheckoutFormSync() function (end of validation)
`
console.log("Validation errors:", errors);
`

WHEN IT FIRES:
- Every time validation completes
- Shows errors that were found during validation

EXPECTED OUTPUT IN CONSOLE (when form is invalid):
`
Validation errors: {
  firstName: "First name is required",
  email: "Please enter a valid email",
  phone: "Phone number must be at least 10 digits"
}
`

EXPECTED OUTPUT (when form is valid):
`
Validation errors: {}
`

## Validation Rules Implemented:

1. ALL FIELDS REQUIRED:
   - firstName, lastName, email, phone, address, city, state, zipCode, country

2. EMAIL VALIDATION:
   - Pattern: standard email format (something@something.something)
   - Error: "Please enter a valid email"

3. PHONE VALIDATION:
   - Minimum: 10 digits (accepts numbers only, removes special characters)
   - Error: "Phone number must be at least 10 digits"

## How to Debug Using Console:

1. Open F12 Console
2. Scroll to "Checkout" page
3. Fill in form fields with test data
4. Click any payment button (PayPal, COD, etc.)
5. Watch for two console.log entries:
   - First shows the form data being validated
   - Second shows any errors found
